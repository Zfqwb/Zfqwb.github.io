  1. 百度git官网，下载链接，根据自己电脑系统下载相应的安装包

  2. 点击下载好的安装包安装这个软件

  3. 一直点击next，直到出现install，点击install，安装完成后点击finish

  4. 安装好后在桌面界面点击鼠标右键，会出现如下界面

  5. 检查git是否安装OK

    • 在桌面或其他地方右键，Git Bash Here

    • 输入git，回车,出现以下界面说明安装成功

  2. 						# 配置用户名
						git config --global user.name "username"    //（ "username"是自己的账户名，）											
					
    						# 配置邮箱

  4. 将ssh文件夹中的公钥（ id_rsa.pub）添加到GitHub管理平台中，在GitHub的个人账户的设置中找到如下界面
  5. 测试一下配置是否成功，在Git Bush命令框（就是刚才配置账号和邮箱的命令框）中继续输入以下命令，回车
    6. 					ssh -T git@github.com

三、Git与Github

  1. 创建本地仓库

    2. 注：需要先创建一个文件夹，然后再这个文件夹内Git Bash Here，再执行 git init

    				$ git init
  2. 把文件添加到仓库：

    3. 注：文件必须在之前的创建的文件夹里，才可以通过命令添加到仓库

    				$ git add readme.md

    Git 命令行添加整个文件夹及目录

    添加整个文件夹及内容

    				git add 文件夹/

    添加目录中所有某种类型的文件

    				git add *.文件类型
  3. 把文件提交到仓库：

    4. git commit命令，-m后面输入的是本次提交的说明，可以输入任意内容，当然最好是有意义的，这样你就能从历史记录里方便地找到改动记录。

    				$ git commit -m “add readme”
  4. git之删除仓库文件

    5. 上传错文件的现象很正常 可能直接使用git add .方便，但也可能传入了一些不想传入的东西 简单的介绍一下删除仓库的文件 

    					git rm -r  你的文件名

    例如 git rm -r README.md

    删除之后需要请求提交

    					git commit -m "remove folder and file list"

    然后再push

    					git push
  5. Git中文件的修改与提交
    • 查看文件版本库中的文件状态：
      • git status

      文件处于未修改状态

    • 修改文件，再次查看版本库中文件的状态：

      • 提示文件已经被修改过，但是还没有进行提交。

    • 提交修改内容与提交新文件是一样的两步
      • git add 文件名
      git commit -m "add a line"
    • 提交后，再次git status查看版本库当前状态，会看到文件没有需要提交的更改。
    • 撤销操作

      • 有时候，我们提交完成后发现漏掉几个文件没有添加，或者信息写错了，这时候就需要对操作进行撤销。

      git commit --amend

      例如 

      													git add file
												
      													git commit -m 'xxxxx'
												
      													git commit --amend

  6. 同步上传到Github

      7. 确定上传的仓库地址

      例如我们给一个叫Test的仓库同步

    				$ git remote add origin    git@github.com:Zfqwb/Zfqwb.github.io.git
      推送到你的仓库 
    				$ git push -u origin master

    注：第一次推送时，我们要添加-u 之后我们可以这条命令

    				$ git push origin master
  7. 从github上下载到本地
    8. 				$ git clone git@github.com:Zfqwb/Zfqwb.github.io.git
  8. 【git】强制覆盖本地代码（与git远程仓库保持一致）

    9. git强制覆盖：

      
    				git fetch --all
			
    				git reset --hard origin/master
			
    				git pull

    git强制覆盖本地命令（单条执行）：

        git fetch --all && git reset --hard origin/master && git pull

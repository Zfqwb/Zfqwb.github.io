多线程补充

AtomicInteger

概述:

​ 是一个原子性的Integer类.
​ 内部的操作可以保证多线程的原子性.

构造:

构造方法 说明
public AtomicInteger(); //默认值为0
public AtomicInteger(int num); //默认值为num

public int get(); //获取值

并发包

概述:

​ 并发包就是一些线程安全的集合类.

分类:

并发包 对应的
ConcurrentHashMap //线程安全的HashMap
CopyOnWriteArrayList //线程安全的ArrayList
CopyOnWriteArraySet //线程安全的HashSet

ConcurrentHashMap

用hashMap会出错

Hashtable(过时的,悲观锁,效率低)
ConcurrentHashMap(推荐,乐观锁,效率高)

CountDownLatch

概述:

​ 在多线程情况下,可以控制线程的执行顺序的类
​ 确定某个线程在其他线程执行之后再执行.

构造:

构造方法 说明
public CountDownLatch(int num); //创建一个对象,指定一个计数器

方法:

方法 说明
public void await(); //当前线程等待,当计数器归零的时候,自动唤醒
public void countDown(); //计数器-1

Semaphore

概述:

​ 可以控制”同时运行的线程的数量”.
​ 和synchronized功能类似
​ 该类允许多个线程同时拥有”锁”的

构造:

​ public Semaphore(int num);
​ //最多允许同时运行num个线程

方法:

​ void acquire();
​ //获取”许可证”,运行线程
​ void release();
​ //释放”许可证”

单元测试

步骤:

1:导入jar包. (模块根目录下新建lib文件夹,右键add as Directory )

​ junit-4.12.jar
​ hamcrest-core-1.3.jar

2:在对应方法上加上@Test注解

注解:

用法 说明
@Test //表示测试该方法
@Before //在测试的方法前运行
@After //在测试的方法后运行

注意:

​ 测试方法必须是”公共的,无返回值的,无参数的”.

日志

步骤:

1:导入log4j的jar包

2:导入配置文件(放在src目录下)

​ log.debug(“debug级别的日志-详细信息”);

1.类的加载器(了解)

作用：

​ 负责将.class文件加载到内存中，并为之生成对应的 java.lang.Class 对象。

分类：

1:BootstrapClassLoader

1.启动类加载器/内置类加载器
2.负责加载$JAVA_HOME中jre/lib/rt.jar里所有的class.
3.JDK中的默认有的类，都由该加载器加载。
4.由C++实现，程序员无法获取也无法操作。

2:ExtensionClassLoader

平台类加载器
负责加载java平台中扩展功能的一些jar包.
包括$JAVA_HOME中“jre\lib*.jar”或-Djava.ext.dirs指定目录下的jar包

3:SystemClassLoader

应用类加载器, 也叫AppClassLoader
负责加载classpath中指定的jar包及目录中class.
就是我们自己写的类都是由该加载器加载

加载配置文件

语法:

InputStream is = 当前类名.class.getClassLoader().getResourceAsStream(“文件名”);
InputStream is = ClassLoader.getSystemClassLoader().getResourceAsStream(“文件名”);

注意:

配置文件必须放在src根目录下.

2.反射

概念：

​ 所谓的反射，指的是，程序在运行过程中，获“类的字节码”文件，再从“类的字节码”中，获取类的“成员变量”，“成员方法”，“构造方法”，并且使用他们，这种现象就叫做反射。

步骤：

  •     1.先获取“类的字节码”

  •     2.从“类的字节码”获取“成员变量”，“成员方法”，“构造方法”

  •      3.使用这些“成员变量”，“成员方法”，“构造方法”

继承&抽象

继承

概述

​ 所谓继承, 和生活中的继承类似, 都是子继承父, 子就拥有了父的财产
​ Java中的继承类似, 子类继承父类, 子类拥有父类的成员

语法

​ public class 子类 extends 父类{ }

注意

​ Java只支持单一继承, 不支持多继承,但支持多层继承
​ 构造方法不能集成

优缺点

优点

​ 提高代码的复用性
​ 其实就是少写代码了
​ 提高代码的维护性

弊端

​ 增加了代码的耦合性
​ 降低了代码的灵活性
​ 继承就只能全部继承, 不允许部分继承

构造方法

​ 子类构造方法执行之前必须先执行父类构造方法
​ 有儿子之前必须先有父亲
​ 子类默认调用父类的无参构造方法
​ 如果想调用父类的有参构造方法,可以使用”super(参数…)”
​ 如果想调用本类的有参构造发方法,可以使用”this(参数…)”, 但调用的最后, 仍然需要调用父类的构造方法

方法重写

概述

​ 当父类中的方法和子类中的方法完全一模一样的时候, 这两个方法之间的关系就是”方法重写”
​ 方法重写也叫”方法覆写. 方法覆盖”

特点

​ 有子父类关系
​ 子类中方法和父类中方法一模一样

场景

​ 当子类对父类方法不满意的时候,就可以重写

注意

​ 1.私有方法不能重写
​ 2.静态方法虽然可以书写,但不属于方法重写
​ 3.构造方法不能重写
​ 4.子类方法的权限必须大于等于父类方法权限

区别

方法重写Override

​ 在子父类中
​ 方法声明一模一样
​ 在方法中, 除了方法体之外的东西, 统称方法声明

方法重载Overload

​ 在同一个类中
​ 方法名相同
​ 参数不同

抽象

关键字

​ abstract

概述

​ 被abstract修饰的类, 被称之为”抽象类”
​ 被abstract修饰的方法, 被称之为”抽象方法”

特点

​ 抽象方法没有方法体
​ 抽象方法所在的类必须是抽象类
​ 子类必须重写抽象类中的所有抽象方法

注意

​ 抽象类不能创建对象
​ 抽象类有构造方法
​ 抽象类中可以没有抽象方法,但抽象方法所在的类必须是抽象类
​ 抽象类的子类
​ 要么也是抽象类
​ 要么就重写抽象类中的所有抽象方法(推荐)

JavaSE进阶-Day01 static

本文最后更新于：4 天前

static

分类思想

​ 概述
​ 一个类的代码太多了,写不下了,就分多个类书写
​ 拆分规则
​ 一般都拆分为三个部分
​ “用户/界面”相关的
​ 界面层/UI层/Web层/Controller层
​ 其他代码,一般是和业务有关的
​ 业务逻辑层/Service层
​ 操作数据,进行增删改查的
​ 数据访问层/Dao层

包

概述

​ 包其实就是文件夹, 当一个包中类太多的时候,可以分包存储

规则

包名全小写字母

一般是公司域名倒着写,并且去掉www

​ com.itheima
​ com.baidu

语法

定义包

语法:

​ package 包名;

位置:

​ 类的最上边

意义:

​ 表示当前类所在的位置

导入包

语法:

​ import 包名.类名;

位置:

​ 类的上边, package的下边

意义:

​ 导包,本质上其实是导入类, 相当于把类拉到了本类内部, 在本类可以直接使用被导入的类.

类之间的访问

相同包

​ 直接访问即可

不同包

​ 先导入包,再访问
​ 通过全类名(“包名 + 类名”), 进行访问

static

概述

​ 表示静态, 可以修饰成员变量. 可以修饰成员方法

作用

​ 被static修饰的成员变量, 所有对象共享(共用一个)
​ 被static修饰的成员方法, 可以通过”类名.静态方法”来访问

注意

​ 静态只能访问静态

场景

​ 当成员变量被所有对象共享时, 可以把成员变量设置为static
​ 当想使用”类名.方法”调用方法时, 可以把成员方法设置为static

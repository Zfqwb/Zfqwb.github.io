XML&枚举

1.XML概述

概念

​ XML是一种可扩展的标记语言.

解析

标记语言

​ 通过标签来描述数据的一门语言

可扩展

标签的名字是可以自定义的
XML文件是由很多标签组成的
而标签名是可以自定义的

2.标签

概述:

由一对<>所构成的语法,就是标签.

例如: 就是student标签

标签有时我们也将其称之为元素

格式:

​ <标签名 属性=”属性值” >标签体(内容)</标签名>

规则:

1:标签必须有开始有结束.

2:标签可以有多个属性,但属性之间使用空格隔开.

3:属性值必须用引号引起来(单双都可以).

4:标签必须正确嵌套,不允许交叉.

3.XML的文档声明:

格式:

​ <?xml version="1.0" encoding="UTF-8"?>

解析:

version :当前xml的版本,必须有,且必须是”1.0”

encoding:当前xml的字符集. 一般为”UTF-8”

4.XML语法规范:

1:文件后缀名必须是xml

2:文档声明必须在文件的第一行第一列.

3:有且仅有一个跟标签

4:特殊符号一般写在CDATA区.

​ <![CDATA[ 内容 ]]>

5.XML的解析的方式

DOM解析:Document Object Model.

一次性将文档的所有元素都加载到内存中.

可以对整个文档进行”增删改查”操作.

SAX解析:Simple Api for XML.

事件驱动的方式,一次加载并解析一行代码.

只能对文档进行”查询”工作

6.XML的解析-DOM4J

步骤:

1:导入dom4j的jar包

2:创建解析器

3:解析文档获得代表文档的Document对象.

4:获得根节点.

5:从跟节点下查找其他的节点.

方法:

​ SAXReader reader = new SAXReader();
​ // 创建解析器
​ Document document = 解析器.read(“文档路径”);
​ // 指定要解析的xml文档
​ Element root = 文档对象.getRootElement();
​ // 获文档得根节点(根节点也是节点)
​ Element e = 节点.element(“标签名”);
​ // 查找节点中的第一指定标签,返回对应节点
​ Element e = 节点.elements(“标签名”);
​ // 查找节点中的所有指定标签,返回节点集合

操作:

​ String elName = 节点.getName();
​ // 获取节点的名字
​ String elText = 节点.getText();
​ // 获取节点中的文本
​ String value = 节点.attributeValue(“属性名”);
​ // 获取节点的指定属性的值

注意:

​ 在查找元素时,只能一层一层的往下找.

9.自定义注解(了解)

定义:

格式:

​

注意:

1:public可以省略.
2:”default 默认值”可以省略
3:数据类型可以是”基本数据类型,String,Class,枚举,注解”,或者是”这几种类型的一维数组”

使用:

语法

@注解名(属性名=属性值,属性名=属性值…)

​ //在类上/方法上/成员变量上使用

注意:

1:注解在使用的时候,必须保证”注解中的每个属性都有值”
2:如果属性有默认值,可以赋值,也可以不赋值,但如果没有默认值,则使用时必须赋值
3:如果使用注解只需要给value属性赋值,则赋值时,value属性名可以省略.
4:如果赋值的属性是数组,且赋值时之赋值了一个值,则赋值时的{}可以省略
5:如果注解的所有属性都有默认值,则使用注解时,可以不带()

意义:

自定义注解本身没有意义,

他的意义在于后续代码为该注解赋予了什么特殊功能.

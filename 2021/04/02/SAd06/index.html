异常

概述

​ 程序发生的不正常现象(报错),就称之为”异常”.
​ 注意, 这里的错误指的不是”语法性错误”

异常体系

Throwable

​ Error: 错误.一般是硬件问题,是没法解决的问题.
​ Exception异常.
​ 运行时异常:
​ 1.运行代码的时候报的错就是运行时异常.
​ 2.运行时异常一般指的是RuntimeException及其子类.
​ 编译时异常:
​ 1.写代码的时候报的错(排除语法错误),就是编译时异常.
​ 2.除了RuntimeException及其子类,其他类都是编译时异常.

注意

​ 通常所说的异常, 指的就是Exception

异常处理

JVM处理异常

​ 1.把”异常类型,异常位置,异常原因”打印到控制台.
​ 2.终止当前程序的运行.

自己处理异常

方式:

​ 异常声明: throws
​ 异常捕获: try-catch

注意:

​ 自己处理异常,目的并不是要把异常干掉
​ 1.把异常信息获取到(类型,原因,位置等)
​ 2.让这个异常不影响其他的代码的正常运行.

throw

​ throw叫做异常抛出
​ 可以理解为”故意制造一个异常”

语法

异常处理

异常声明 throws

语法

​ public void 方法名() throws 异常类型{ … }

位置:

​ 方法参数后边.

意义:

​ 告诉方法的调用者,我很危险.
​ 一般”编译时异常”都使用该方式处理.

异常捕获 try-catch

语法:

​

执行流程

​ 1.如果try中不出现异常,则执行完try之后,整个try-catch结束, catch中的代码不执行
​ 2.如果try中出现异常,则立即跳转到catch中执行,catch执行完毕后,整个try-catch结束
​ 3.当try中出现异常的时候, JVM会把异常捕获,并做成一个异常对象, 交给catch()中的异常变量

注意事项

​ 1.catch()中的异常类型,必须和捕捉到的异常类型一致.
​ 2.catch可以有多个.
​ 3.catch()中的异常类型一般直接写为Exception

意义:

​ 对于可能出现异常的代码进行封装,让其不影响其他代码的执行.
